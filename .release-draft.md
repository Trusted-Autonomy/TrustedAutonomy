## v0.13.17-alpha.7

### New Features

- **Supervisor agent**: Goals can now be reviewed by a supervisor agent that checks alignment with your product constitution before a draft is approved (`ta goal start --supervisor`)
- **Supervisor authentication**: Supervisor agents support credential-scoped auth tokens for multi-user and team workflows
- **Multi-agent goal support**: Goals can now coordinate across multiple concurrent agents with shared context
- **VCS environment isolation**: Spawned agents run in a clean VCS environment — git credentials and local config no longer leak into agent subprocesses
- **Community Hub redesign**: Plugin discovery and community context sources have a redesigned hub interface with improved browsing and installation flow

### Improvements

- **`Finalizing` state now visible**: Goals in the `Finalizing` phase are correctly shown in `ta goal list` rather than appearing stale
- **Draft safety checks**: `ta draft apply` now validates the staging workspace before applying to prevent partial or corrupted applies
- **GC cleanup**: Garbage collection now removes orphaned staging directories more reliably after failed or cancelled goals
- **Branch prefix config**: Feature branch prefix is now configurable (default `feature/`) via team config
- **Release asset pipeline**: Release builds now use a draft-first workflow to avoid asset upload failures on immutable releases

### Bug Fixes

- Fixed `--verbose` flag not being passed through to the supervisor agent process
- Fixed phase IDs with a `v` prefix (e.g. `v0.13.17.3`) being rejected — the `v` is now normalised automatically

_Changes since v0.13.1-alpha.7_
