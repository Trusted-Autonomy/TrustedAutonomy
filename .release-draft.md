## v0.9.9-alpha.5

### New Features

- **Workflow & Agent Authoring Tooling** — New `ta workflow new` and `ta agent new` commands for scaffolding workflow definitions and agent configurations from built-in templates. `ta workflow validate` and `ta agent validate` provide comprehensive pre-flight checks with actionable feedback on schema errors, undefined roles, dependency cycles, and missing agent configs. Includes an example library with 5 workflow templates, 4 agent templates, and 4 version schema templates.
- **External Channel Delivery** — Draft reviews and agent questions can now be delivered to Slack, Discord, and Email. New `ChannelDelivery` trait with per-channel adapters, `ChannelDispatcher` for daemon-side routing, and `channels` routing field on `AgentNeedsInput` events. Configure channels in `daemon.toml`.
- **`ta plan from <doc>` Wrapper** — New convenience command that reads a product document and launches an interactive agent session to generate a phased PLAN.md. Three-tier document resolution (configurable folder patterns, tree walk, error with guidance) and project-configurable search directories via `.ta/plan-schema.yaml`.
- **Shell TUI Interactive Mode** — Agent questions now surface inline in `ta shell`, so humans can see and respond without leaving the TUI. New `ta conversation` command for viewing Q&A history from past sessions.
- **Interactive Mode Core Plumbing** — New `ta_ask_human` MCP tool with file-based signaling enables agents to pause, ask clarifying questions, and receive human responses during goal execution.

### Improvements

- **Configurable Commit Co-Authorship** — Commit co-author trailers are now configurable via `workflow.toml` instead of being hardcoded. Defaults updated for the renamed `trustedautonomy-agent` GitHub account.
- **Plan Phase Additions** — Added roadmap phases for event-driven agent routing (v0.11.0), SourceAdapter unification & `ta sync` (v0.11.1), and BuildAdapter & `ta build` (v0.11.2).
- **Mission & Scope Documentation** — Added framing document describing TA's mission, scope, and product positioning.
- **Interactive Mode Sub-Phase Structure** — Restructured the interactive mode milestone into sub-phases (v0.9.9.1–v0.9.9.4) for clearer incremental delivery and tracking.

### Bug Fixes

- **Release Pre-Release Default** — Fixed the default pre-release suffix used during release pipeline execution.
