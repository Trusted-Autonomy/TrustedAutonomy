## v0.16.6-alpha.1

### New Features

- **VS Code Extension** — TA is now available as a VS Code extension
- **JetBrains IDE Plugin** — support for PyCharm, WebStorm, and IntelliJ IDEA
- **Neovim Plugin** — Neovim integration for terminal-first development workflows
- **Ollama Agent Plugin** — run agent goals using local models via the Ollama plugin
- **Gemma 4 Agent Profiles** — built-in agent profiles for Gemma 4 models via Ollama
- **Studio Advisor Engine** — advisor functionality now available in the Studio web UI
- **Agent Context Files & Manifest Inheritance** — agents can declare and inherit context files via their manifests
- **Windows Sandboxing** — agent processes are isolated using Job Objects and AppContainer on Windows
- **Windows ProjFS** — Projected File System support with an onboard wizard step to enable it
- **Template Engine** — data-driven goal templates with composable feature components
- **Cross-Project TA Links** — connect related TA projects via `.ta/links.toml`
- **`ta plan init --discover`** — agent-assisted plan discovery with per-question lookup
- **`ta style`** — project-wide developer style constitution
- **`ta doctor` Intelligence** — runtime validation now includes per-check diagnostics and actionable next steps
- **IDE Index Exclusion** — `.ta/` runtime directories are excluded from IDE indexing automatically via `ide-excludes.json`

### Improvements

- Configured MCP servers are now auto-approved for agent goals — no per-goal approval prompts
- Agent build cache is redirected to the source project directory, avoiding redundant recompilation across goals
- Staging directories are automatically cleaned up when a draft is denied
- Discord plugin reconnects automatically when the daemon restarts (adopt-orphan), preventing crash loops
- Goal titles now have ANSI escape sequences stripped and are truncated to GitHub's 256-character limit
- Studio: sub-phase display, inline draft apply, PR-ready badge, and plan phase navigation

### Bug Fixes

- Terminal state is restored correctly when a PTY agent session exits or is killed
- MCP allow rules no longer accept bare `mcp__*` wildcards; global settings are now merged properly
- Neovim plugin: security fix applied as follow-up to initial ship
- VCS initialization: resolved gaps in LOCAL_TA_PATHS detection and config mirroring for non-git setups
- Studio: fixed draft detail crash, draft filter, and plan phase UX
- Onboard wizard: summary screen now shows the correct dynamic step title
- Plan phase resolution now correctly handles done-phase titles with no pending sub-phases

_Changes since v0.15.30-alpha.7_
