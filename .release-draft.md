## v0.4.5-alpha

### New Features

- **Behavioral Drift Detection** — Detect when an agent's behavior diverges from its historical baseline. Tracks resource scope drift, escalation frequency changes, rejection rate shifts, change volume anomalies, and dependency pattern shifts. New `ta audit drift` command provides per-agent and cross-agent drift reports aligned with NIST AI RMF risk monitoring.
- **Access Constitutions** — Human-authorable per-goal access contracts that declare which URIs an agent should touch. Deviations from the constitution feed into drift detection as high-signal indicators. New `ta goal constitution` subcommands for viewing, setting, proposing, and listing constitutions. Supports automatic constitution proposal from agent baseline patterns.
- **Interactive Session Completion** — PTY-based real-time agent output streaming with mid-session human guidance. Stdin interleaving routes user input through the ReviewChannel; all interactions are audit-logged. Pause/resume support via channel disconnect/reconnect and `ta run --resume <session-id>`.

### Improvements

- **Partial ID matching** — All `ta draft`, `ta goal`, and `ta session` commands now accept 8+ character UUID prefixes instead of requiring full UUIDs.
- **Apply on PendingReview** — `ta draft apply` works directly on PendingReview drafts without requiring a separate `ta draft approve` step first (auto-approves on apply).
- **Plan phase in `ta release run`** — Accepts plan phase IDs (e.g., `0.4.1.2`) and auto-converts to semver release versions (`0.4.1-alpha.2`) via configurable `version_policy` in `.ta/release.yaml`.
- **Terminal encoding safety** — Disposition badges and status markers render cleanly across all terminal encodings.
- **Version tracking & README badges** — README now displays current development and released version badges.
- **USAGE.md rewrite** — User guide rewritten as a consumer-friendly onboarding document with quick-start workflow, configuration reference, and troubleshooting section.

### Bug Fixes

- **Stale CLAUDE.md in `--follow-up`** — Fixed `--follow-up` injecting an outdated CLAUDE.md into follow-up sessions; draft IDs are now accepted correctly in follow-up commands.
