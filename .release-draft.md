## v0.15.30-alpha.7

### New Features

**Goal context & human-in-the-loop**
- `ta run --context <path>` — supply a file as structured goal context instead of inline text
- Live human interjection during agent runs via `ta advise`
- 3-option approval gate: approve, deny, or send reviewer feedback back to the agent
- Project-scoped memory at `.ta/project-memory/` persisted across goals
- `ta init` now generates a starter `CLAUDE.md` for new projects
- Post-install onboarding wizard (`ta onboard`)
- Terms acceptance gate on first-run operations

**Workflows**
- Hierarchical workflows: sub-workflow steps, serial chaining, and parallel fan-out
- Orchestrated work planner + implementor agent split for large goals
- Parameterized workflow templates with runtime variable injection
- Workflow template library: install, publish, and search shared templates
- Intent resolver: describe a goal in natural language and TA selects the right workflow
- Multi-agent consensus review workflow with audit persistence
- Auto-approve constitution: rule-based approval policies with amendment flow
- Batch build loop for automated multi-phase rollouts
- PR CI failure recovery workflow

**Notifications**
- System health notifications in CLI and TA Studio ambient alerts
- Workflow event bus with subscription core
- Notification rules engine: Immediate, Threshold, Schedule, Debounce, and Expiry rules
- Delivery channels: Terminal, Webhook, Email, Slack

**TA Studio (UI)**
- Multi-project support with project browser and platform launchers
- Plan tab: phase browser, one-click run, and custom goal entry
- Workflows tab with agent personas and new project wizard
- Studio advisor / QA agent with source-verification and plan auto-correction
- Studio cleanup button for stale goals and drafts

**Connectors**
- ComfyUI inference connector (`ta connector start comfyui`)
- Unity connector with scene query, test-run, and render-capture MCP tools
- Unreal Engine connector with MRQ governed render flow
- Social media plugins (`ta-connectors/social`)
- Email assistant workflow with MessagingAdapter plugin architecture

**Agent safety & security**
- Agent sandboxing and process isolation
- Security level profiles (Low / Mid / High) for the supervisor
- Supervisor file-inspection mode for security review
- Language-aware static analysis with agent self-correction loop
- Governed interactive session (advisor agent)
- Multi-party approval and threshold governance

**Assets & artifacts**
- Generic binary, text, image, and video artifact support in changesets
- Agent-run contextual asset diffs surfaced in draft review

**Windows**
- ProjFS-based virtual staging workspace on NTFS
- Windows code signing with EV certificate and CI integration

**Auth, extensibility & release**
- `ta doctor` auth validation and agent-agnostic auth spec
- Project TA version tracking and upgrade path
- Auth plugin surface for custom authentication providers
- Daemon extension surface for third-party plugins
- Pluggable memory backends via external plugin protocol
- One-command release (`ta release run`) with phase auto-detection
- Nightly build pipeline
- crates.io publishing infrastructure with rate-limit-aware incremental publish
- Local agent profiles for Qwen3.5 and Gemma via Ollama
- VCS event hooks: inbound webhook and trigger integration
- Local audit ledger (`ta audit`) with full goal traceability

### Improvements

- Apply UX: closing summary, verify-progress header, and animated spinner
- Phase resolution: full version parsing and smart matching in `ta release`
- `ta draft apply` branches from `origin/main` instead of potentially stale local HEAD
- PLAN.md integrity enforcement: pre-apply structural validation and corruption guard
- Reviewer agent uses heartbeat-based liveness instead of wall-clock timeout
- Phase in-progress marking at goal start for accurate status reporting
- Phase claim locking prevents duplicate workflow dispatch
- All internal git calls routed through the VCS adapter layer (no raw `git` exec)
- `ta gc` absorbed into `ta doctor --fix`; `ta gc` remains as a non-interactive alias
- Supervisor heartbeat stale threshold raised from 30 s to 90 s
- `ta draft close` command to dismiss drafts without applying
- Inline draft build for interactive CLI sessions
- `ta doctor` version-mismatch messages now suggest the exact corrective command
- `workflow.local.toml` is the canonical override file with deep TOML merge support
- Embedded patches in draft packages enable staging-free apply after GC
- Velocity stats: rework tracking, auto-migration, version filtering, shell and Studio surfaces
- Critical files (goal audit, plan history) auto-staged on `ta draft apply`
- Background process lifecycle with heartbeat and event notification on completion
- Goal shortref unified across goal → draft → PR → audit trail
- Context budget trimming in injected CLAUDE.md reduces token overhead

### Bug Fixes

- MSI installer now correctly upgrades when only the sub-patch version changes
- ProjFS gated behind a Cargo feature flag — no startup crash on Windows without Client-ProjFS
- Added ETXTBSY retry/backoff for overlayfs-backed temp dirs on Nix CI and GH Actions Ubuntu
- Approval gate auto-fails in non-TTY environments instead of hanging indefinitely
- Release tests no longer hang on TTY stdin when run from a terminal
- `ta draft apply` working-tree check now clears TA injection residue before diffing
- PR URL capture handles indented `gh pr list` output correctly
- Onboarding first-run gate skipped when `ANTHROPIC_API_KEY` is already set
- Velocity history committed to the feature branch (no longer lost on `git checkout`)
- Subcrate internal dependency versions updated atomically when workspace version is bumped
- Supervisor prompt delivered via stdin when `--allowedTools` is active
- Decision log no longer bleeds across goals in legacy agent sessions
- PLAN.md phase items preserved on `ta draft apply` when protected-file mtime heuristic misfired
- `mcp__ta__*` tools auto-allowed in injected Claude Code settings to prevent permission prompts
- Truncation panic eliminated on multi-byte (non-ASCII) characters in plan summaries
