## v0.13.12-alpha — Beta Bug Bash & Platform Polish

### New Features

- **Feature Velocity Stats** (`ta stats velocity`): Track goal outcomes over time — applied/denied/cancelled/failed counts, avg and p90 build times. `ta stats export` outputs JSON or CSV.
- **Platform Installers**: macOS DMG and Windows MSI included in GitHub release assets (MSI auto-adds `ta` to system PATH, Start Menu shortcut).
- **Product Constitution Framework** (`ta constitution init`): Draft a behavioral contract for AI agents in `.ta/constitution.md` from your PLAN.md and CLAUDE.md. `ta constitution check` validates §4 inject/restore balance.
- **Agent Framework** (v0.13.8): Pluggable agent backends — Anthropic, OpenAI, Ollama, custom. `ta agent add/list/remove` workflow.

### Bug Fixes

- `ta draft apply` now validates the staging workspace exists before opening it. If deleted by concurrent GC, gives actionable error with exact recovery commands.
- Lock file reminder after apply: `hint:` message prompts to commit `Cargo.lock`, `package-lock.json`, etc. alongside the feature branch.
- Fixed Windows path separator in constitution scan exclude patterns (`src/tests/` was not excluded on Windows).

### Configuration

- Split `stale_hint_days` (default: 3) from `stale_threshold_days` (default: 7) in `[gc]` section. Eliminates false-alarm where startup hint fired for days 3–6 but `ta draft list --stale` found nothing.

### Diagnostics

- `ta --startup-profile <command>` prints wall-clock timing for each startup phase (useful on slow Windows cold-starts).

### Upgrade notes

- Bump `ta` binary. Run `bash install_local.sh` to rebuild from source, or download the binary for your platform below.
- Windows MSI installer now available — installs `ta` and `ta-daemon` to `%ProgramFiles%\TrustedAutonomy\` and adds to PATH.
